공유 가이드 — agentpod export

세션 중에 설치한 플러그인/스킬, 붙인 MCP 서버를 프로젝트 파일로 스냅샷해서 팀원과 git으로 공유하는 방법.

agent.toml .mcp.json 시크릿은 .env로 분리
개요 ① 사용법 ② 무엇을 기록하나 ③ 시크릿 처리 ④ 받는 사람 쪽 ⑤ 프로파일 문제 해결

0개요

세션 중에 /plugin install이나 claude mcp add로 즉석에서 설치·연결한 것들은 프로젝트 폴더(git)가 아니라 호스트 쪽 상태에만 남습니다. 프로젝트별로 분리되어 있어서, 다른 사람이 git clone해도 자동으로 따라오지 않습니다.

agentpod export는 지금 이 프로젝트에 설치·연결된 것들을 프로젝트 파일로 스냅샷해서, 커밋 한 번으로 팀원 전체에 공유되게 해줍니다.

1사용법

# 프로젝트 폴더에서
$ agentpod export
agent.toml: added skill(s) my-tool
.mcp.json: added MCP server(s) jira-mcp (secrets written to .env)
Review the diff, then commit agent.toml / .mcp.json. Never commit .env.

이미 agent.toml/.mcp.json에 있는 항목은 다시 안 씁니다 — 여러 번 실행해도 안전합니다(멱등).

2무엇을 기록하나

대상기록되는 곳
설치+활성화된 플러그인/스킬agent.toml의 [[skills]] (스키마: 스킬 가이드)
claude mcp add로 추가한 MCP 서버.mcp.json의 mcpServers

이미 기본으로 제공되는 소수의 내장 스킬은 굳이 다시 기록하지 않고 건너뜁니다 — agent.toml은 이 프로젝트가 추가로 필요로 하는 것만 담습니다.

3시크릿 처리 — 절대 커밋 파일에 안 박힘

MCP 서버의 env/헤더 값(토큰·API 키 등)은 .mcp.json에 그대로 적히지 않습니다. 대신:

  1. 값을 ${변수명} 플레이스홀더로 치환

    예: "API_KEY": "실제토큰" → "API_KEY": "${JIRA_MCP_API_KEY}"

  2. 실제 값은 .env에 저장

    .env는 레포 전체에서 이미 .gitignore 처리되어 있습니다(agents/n8n/.mcp.json과 동일한 기존 방식). 변수명은 서버 이름을 붙여 충돌을 피합니다(JIRA_MCP_API_KEY).

확인 습관: export 후 git status로 .env가 안 보이는지(=제대로 무시되는지) 한 번 확인하고, git add -f로 강제 추가하지 않도록 주의하세요.

4받는 사람 쪽 — git clone + agentpod run이 끝

git diff agent.toml .mcp.json으로 뭐가 추가됐는지 확인하고 커밋·푸시하면, 팀원은:

$ git clone ...
$ agentpod run

만 실행하면 됩니다. 부팅 시 agent.toml의 스킬이 자동 설치되고, .mcp.json의 MCP 서버가 자동 승인됩니다.

단, 시크릿 값은 각자 채워야 함: .mcp.json의 ${VAR}가 참조하는 실제 값은 export가 대신 채워주지 않습니다 — 값 자체가 시크릿이라 공유 대상이 아니기 때문입니다. 팀원 각자 자기 .env에 그 변수를 채워야 MCP 서버가 실제로 동작합니다.

5여러 프로젝트/계정을 일부러 공유하고 싶다면

agentpod run --profile 이름처럼 프로젝트마다 같은 --profile 이름을 쓰면 그 프로젝트들끼리는 플러그인 설치 상태를 공유합니다(로그인은 프로필과 무관하게 항상 공유). agentpod export --profile 이름으로 그 프로필의 상태를 내보낼 수도 있습니다.

6문제 해결

증상원인 / 해결
agentpod export가 아무것도 안 씀이미 agent.toml/skills.toml에 선언돼 있거나, 기본 제공 스킬뿐이거나, 아직 아무것도 설치·연결한 게 없는 경우입니다.
export한 MCP 서버가 팀원 쪽에서 안 붙음.mcp.json의 ${VAR}가 참조하는 실제 값을 팀원 각자 .env에 채워야 합니다 — 값 자체는 시크릿이라 export가 대신 채워주지 않습니다.
다른 프로젝트에서 설치한 게 여기 export에 안 보임정상입니다 — 플러그인 설치 상태는 프로젝트별로 분리됩니다. 같은 걸 공유하려면 §5의 --profile을 쓰세요.
