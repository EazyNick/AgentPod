Bitbucket 봇 계정 SSH 등록

봇 전용 SSH 키를 호스트 ~/.agent/ssh에 한 번 만들어 Bitbucket에 등록하면, 이후 모든 에이전트 컨테이너가 그 키로 clone·push 합니다. AgentPod가 openssh-client·~/.ssh 마운트·GIT_SSH_COMMAND를 자동 처리합니다.

호스트 1회 등록 전 컨테이너 공유 SSH 키 (토큰 아님)
개요 ① 봇 계정 ② 키 생성 ③ Bitbucket 등록 ④ 커밋 작성자 ⑤ 실행 ⑥ 확인 문제 해결

0개요

인증 방식은 HTTPS 토큰과 SSH 키 두 가지가 있습니다. 이 가이드는 Bitbucket에서 흔한 SSH 키 방식입니다. 키를 호스트 ~/.agent/ssh에 두면 로그인·크레덴셜과 동일하게 모든 컨테이너가 공유합니다. 컨테이너 안에서 따로 설정할 필요가 없습니다.

AgentPod 자동 처리: 이미지에 openssh-client 포함, ~/.agent/ssh가 있으면 /home/agent/.ssh로 마운트, 봇 키가 있으면 entrypoint가 GIT_SSH_COMMAND(키 지정·known_hosts·accept-new)를 설정.

1봇 계정 준비

커밋 히스토리에서 사람과 봇을 구분하려면 전용 봇 Bitbucket 계정 권장. 공개키를 어디에 등록하느냐로 접근 범위가 갈립니다:

등록 위치범위권장 상황
계정 SSH 키 (Personal settings)그 봇 계정이 접근 가능한 모든 repo봇이 여러 repo를 다룰 때
저장소 Access key (deploy key)그 repo 하나 (Read 또는 Write)최소 권한 · repo 단위 격리

2SSH 키 생성 (호스트, 1회)

$ mkdir -p ~/.agent/ssh && chmod 700 ~/.agent/ssh
$ ssh-keygen -t ed25519 -C "agent-bot" -N "" -f ~/.agent/ssh/id_ed25519
$ chmod 600 ~/.agent/ssh/id_ed25519
  • -N "" : 패스프레이즈 없음(무인 실행용). 보안 강화하려면 패스프레이즈 + ssh-agent 필요(무인엔 부적합).
  • 생성물: 개인키 id_ed25519, 공개키 id_ed25519.pub.
권한 필수: ~/.agent/ssh=700, 개인키=600. 소유자는 uid 1000(리눅스/WSL2 기본 사용자 = 컨테이너 agent)이어야 ssh가 거부하지 않습니다.

3공개키를 Bitbucket에 등록

공개키 내용을 복사합니다:

$ cat ~/.agent/ssh/id_ed25519.pub

  1. 계정 키로 등록

    Bitbucket 우측 상단 아바타 → Personal settings → SSH keys → Add key → Label 입력, 공개키 붙여넣기 → Save.

  2. 또는 저장소 Access key(deploy key)로 등록

    대상 저장소 → Repository settings → Access keys → Add key → 공개키 붙여넣기. push가 필요하면 write 권한 옵션을 켭니다.

계정 키 vs deploy key: 여러 repo에 push하는 봇이면 계정 키가 편하고, 딱 한 repo만 다루면 deploy key(최소 권한)가 안전합니다.

4커밋 작성자 설정

SSH는 인증(push/pull)만 담당합니다. 커밋에 찍히는 작성자는 따로 설정합니다(호스트 1회, 공유):

# agentpod CLI 있으면 (토큰은 SSH라 불필요)
$ agentpod git-setup --name "agent-bot" --email "봇-Bitbucket-이메일"

# CLI 없이 docker만이면 ~/.agent/gitconfig 직접 작성
$ cat > ~/.agent/gitconfig <<'EOF'
[user]
	name = agent-bot
	email = 봇-Bitbucket-이메일
[safe]
	directory = *
EOF

이메일은 커밋이 봇 계정에 귀속되도록 봇의 Bitbucket 이메일로 맞추세요.

5실행 — SSH 리모트 사용

# agentpod CLI: ~/.agent/ssh 가 자동 마운트됨
$ cd 프로젝트 && agentpod run

# docker만: ssh 디렉토리를 직접 마운트
$ docker run --rm -it -v "$PWD":/project -w /project \
    -v ~/.agent/claude:/home/agent/.claude \
    -v ~/.agent/gitconfig:/home/agent/.gitconfig:ro \
    -v ~/.agent/ssh:/home/agent/.ssh \
    agentpod:latest claude --dangerously-skip-permissions

리모트는 SSH URL이어야 합니다(HTTPS가 아니라):

agent@…$ git remote set-url origin git@bitbucket.org:WORKSPACE/REPO.git
# 새로 clone 할 때도 SSH URL
agent@…$ git clone git@bitbucket.org:WORKSPACE/REPO.git

6확인

agent@…$ ssh -T git@bitbucket.org
# → "authenticated via ssh key ... You can use git ..." 류 메시지면 성공
agent@…$ echo "$GIT_SSH_COMMAND"     # 키 지정이 들어가 있어야 함
agent@…$ git config --global user.name  # agent-bot
agent@…$ git push

7문제 해결

증상원인 / 해결
Permission denied (publickey)공개키가 Bitbucket에 등록 안 됨 / 다른 키 사용. id_ed25519.pub를 계정 또는 deploy key로 등록했는지 확인. ssh -T git@bitbucket.org -v로 어떤 키를 쓰는지 확인.
Host key verification failedknown_hosts에 bitbucket.org 없음. 호스트에서 ssh-keyscan bitbucket.org >> ~/.agent/ssh/known_hosts (entrypoint는 accept-new라 최초 접속 시 자동 추가되지만, ~/.ssh가 쓰기 가능해야 함).
Bad owner or permissions / 키 무시됨ssh 권한 엄격. chmod 700 ~/.agent/ssh, chmod 600 id_ed25519, 소유 uid 1000 확인.
여전히 HTTPS로 나감리모트가 HTTPS URL. git remote set-url origin git@bitbucket.org:WS/REPO.git로 SSH URL 전환.
write 거부(deploy key)deploy key를 read-only로 등록함. Bitbucket에서 해당 Access key에 write 권한 부여.
커밋 작성자가 봇이 아님§4의 user.email을 봇 Bitbucket 이메일로 설정.
GitLab / GitHub도 동일: 호스트를 gitlab.com/github.com으로 바꿔 공개키 등록 + SSH URL(git@github.com:...) 사용. 컨테이너 설정은 그대로.
