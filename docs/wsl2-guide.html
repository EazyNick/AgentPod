AgentPod · WSL2

WSL2 사용자 가이드 설치 포함

Windows + WSL2(Ubuntu) 환경에서 AgentPod를 네이티브로 설치하고 실행하는 전 과정. WSL2 설치 → Docker 연동 → Python 준비 → AgentPod 설치 → 첫 실행까지 순서대로 따라 하면 됩니다.

Windows 11 + WSL2 Ubuntu 배포판 Docker Desktop (WSL 백엔드) Python 3.12+
왜 WSL2 ① WSL2 설치 ② Docker 연동 ③ Python 준비 ④ AgentPod 설치 ⑤ 이미지 빌드 ⑥ 첫 실행 동작 확인 문제 해결

0왜 WSL2가 권장 타깃인가

AgentPod의 오케스트레이터는 리눅스 기본 동작(os.kill 기반 PID 검사, POSIX 시그널, ~/.agent 경로, uid 정합)을 전제로 설계되었습니다. WSL2 안의 Ubuntu는 진짜 리눅스 커널을 쓰므로 이 모든 것이 깔끔하게 동작합니다. 또한 WSL2 기본 사용자 uid가 1000이라, 컨테이너의 agent(uid 1000)와 맞아 바인드 마운트 파일 권한이 자연스럽게 정합됩니다.

준비물 체크리스트
  • Windows 10 22H2+ 또는 Windows 11
  • 관리자 권한 (WSL 최초 설치 시)
  • Docker Desktop 설치 가능 환경

1WSL2 + Ubuntu 설치

이미 Ubuntu 배포판이 있다면 이 단계는 건너뛰어도 됩니다. (확인: PowerShell에서 wsl -l -v → Ubuntu가 보이고 VERSION이 2)

  1. WSL 설치 (PowerShell 관리자)

    PS> wsl --install -d Ubuntu

    재부팅이 요구될 수 있습니다. 설치 후 Ubuntu가 처음 실행되면 리눅스 사용자 이름/비밀번호를 만듭니다(이 사용자가 uid 1000이 됩니다).

  2. WSL2인지 확인

    PS> wsl -l -v
  NAME              STATE      VERSION
  Ubuntu            Running    2      ← VERSION이 2여야 함

    만약 1이면: wsl --set-version Ubuntu 2

  3. Ubuntu 진입 & 최신화

    PS> wsl -d Ubuntu
$ sudo apt update && sudo apt upgrade -y

2Docker 연동 (핵심)

AgentPod는 docker 명령을 Ubuntu 안에서 호출합니다. Docker Desktop의 WSL 통합을 켜면 Ubuntu에서 같은 Docker 엔진을 쓸 수 있습니다.

  1. Docker Desktop 설치 & WSL2 백엔드

    Docker Desktop을 설치하고 실행합니다. Settings → General에서 “Use the WSL 2 based engine”이 켜져 있는지 확인합니다.

  2. Ubuntu에 통합 켜기

    Docker Desktop → Settings → Resources → WSL Integration → Ubuntu 토글을 ON → Apply & restart.

    이 토글이 꺼져 있으면 Ubuntu 안에서 docker: command not found 또는 “could not be found in this WSL 2 distro” 오류가 납니다.

  3. Ubuntu에서 docker 확인

    $ docker version --format '{{.Server.Version}}'
# 서버 버전이 출력되면 성공
$ docker run --rm hello-world

3Python 준비

AgentPod 오케스트레이터는 Python 3.12+가 필요합니다. Ubuntu의 Python이 pip/venv 없이 최소 설치된 경우가 있으니 함께 설치합니다.

$ python3 --version            # 3.12 이상인지 확인
$ sudo apt install -y python3-pip python3-venv git
가상환경(venv) 권장. 시스템 파이썬을 어지럽히지 않도록 프로젝트 전용 venv를 씁니다. 
$ python3 -m venv ~/.venvs/agentpod
$ source ~/.venvs/agentpod/bin/activate
# 프롬프트 앞에 (agentpod)가 붙습니다
파일 위치 팁: 소스를 /mnt/c/...(윈도우 파일시스템)보다 ~/(리눅스 파일시스템) 아래 두면 성능·권한 문제가 없습니다.

4AgentPod 설치

  1. 소스 가져오기

    $ cd ~
$ git clone <repo-url> AgentPod      # 또는 기존 소스를 복사
$ cd AgentPod

    Windows 쪽에 이미 소스가 있다면 리눅스로 복사: cp -r /mnt/c/Users/<you>/Downloads/Project/AgentPod ~/AgentPod

  2. 패키지 설치

    $ pip install -e "."
# 개발/테스트까지
$ pip install -e ".[dev]"

  3. 설치 확인 & 유닛 테스트

    $ agentpod --help
$ pytest -m "not integration"
# 33 passed 가 나오면 정상
설치 후엔 소스 폴더와 무관하게 어디서나 실행됩니다. agentpod는 venv 안에 설치되므로, venv가 활성화된 셸이면 어느 디렉토리에서든 쓸 수 있습니다. ~/dev/AgentPod로 돌아갈 필요가 없습니다 — agentpod run은 현재 있는 폴더(cwd)를 프로젝트로 간주합니다. 
# 새 터미널을 열 때마다 venv 활성화 한 번
$ source ~/.venvs/agentpod/bin/activate
(agentpod) $ cd ~/work/my-project     # 아무 프로젝트
(agentpod) $ agentpod run             # 그 폴더 기준 컨테이너
매번 활성화가 번거롭다면 ~/.bashrc 끝에 source ~/.venvs/agentpod/bin/activate 한 줄을 넣어 자동화할 수 있습니다.

5에이전트 이미지 빌드

최초 1회. ubuntu·Node·claude 설치로 몇 분 걸리며, 이후에는 캐시로 빨라집니다.

$ agentpod build
# 완료 후 확인
$ docker run --rm agentpod:latest bash -lc "claude --version && whoami && id -u"
# 예: 2.1.199 (Claude Code) / agent / 1000

6첫 실행

  1. 프로젝트로 이동

    $ cd ~/work/my-project

  2. 인증 준비 (택1)

    API 키: 프로젝트 루트에 .env 생성

    # ~/work/my-project/.env
ANTHROPIC_API_KEY=sk-ant-...

    또는 대화형 로그인: 실행 후 컨테이너 셸에서 claude login

    $ agentpod shell
agent@…$ claude login
agent@…$ exit

  3. (선택) 컨테이너별 지침 넣기

    $ agentpod context          # 컨텍스트 폴더 경로 출력
$ mkdir -p "$(agentpod context | head -1)"
$ $EDITOR "$(agentpod context | head -1)/CLAUDE.md"

  4. 실행

    $ agentpod run
# 격리된 컨테이너 안에서 Claude Code 대화형 세션 진입

  5. 종료 & 자동 정리

    세션을 빠져나오면(Ctrl+D) 마지막 세션일 때 컨테이너가 자동으로 멈춥니다.

7동작 확인 (스모크 테스트)

컨테이너가 제대로 뜨고 Claude가 준비됐는지 비대화형으로 확인:

$ cd ~/work/my-project
$ agentpod run &            # 또는 다른 터미널에서
$ agentpod status           # agent-… 컨테이너가 running 인지

# 컨테이너 이름을 확인해 내부 상태 점검
$ C=$(docker ps --filter name=agent- --format '{{.Names}}' | head -1)
$ docker exec "$C" bash -lc 'whoami; id -u; claude --version; cat /home/agent/.claude/CLAUDE.md 2>/dev/null'
# agent / 1000 / 2.1.199 (Claude Code) / @/home/agent/context/CLAUDE.md (컨텍스트 넣었다면)
확인 항목기대값
컨테이너 상태running
사용자 / uidagent / 1000
Claude Code버전 문자열 출력 (예: 2.1.199)
MD 주입@/home/agent/context/CLAUDE.md (컨텍스트 폴더가 있을 때)

8WSL2 특유의 문제 해결

증상해결
Ubuntu에서 docker: command not foundDocker Desktop → Settings → Resources → WSL Integration에서 Ubuntu 토글 ON → Apply & restart.
ensurepip is not available (venv 실패)sudo apt install -y python3-venv 후 다시 python3 -m venv.
No module named pipsudo apt install -y python3-pip.
기본 배포판이 docker-desktop정상(도커용 유틸 VM). AgentPod는 wsl -d Ubuntu의 Ubuntu에서 실행하세요.
Docker 데몬 미기동Windows에서 Docker Desktop 앱을 실행(트레이 아이콘 초록).
/mnt/c 경로에서 느림/권한 경고소스를 리눅스 홈(~/)으로 옮겨 실행.
마운트된 프로젝트 파일 쓰기 오류WSL2 기본 사용자 uid가 1000인지 확인(id -u). 컨테이너 agent가 uid 1000이라 일치해야 함.

9명령어 요약

$ agentpod build [--force]   # 이미지 빌드
$ agentpod run              # 컨테이너 스폰/재사용 → Claude 대화형
$ agentpod shell            # 컨테이너 bash (로그인·디버깅)
$ agentpod status           # 전체 컨테이너·세션 수
$ agentpod stop             # 현재 프로젝트 컨테이너 stop
$ agentpod rm               # stop + 제거
$ agentpod context          # MD 컨텍스트 폴더 경로
