봇 git 신원 등록 가이드

자율 에이전트가 사람 계정이 아닌 전용 봇 신원으로 커밋·푸시하도록 설정합니다. 한 번 등록하면 모든 컨테이너가 공유합니다. 커밋 히스토리에서 "봇이 한 일"이 명확히 구분되고, 사람 개인키를 에이전트에 노출하지 않습니다.

등록은 호스트에서 컨테이너는 자동 적용 한 번 = 전 컨테이너 공유
왜 봇 신원 ① PAT 발급 ② 호스트 등록 (권장) ③ 환경변수 (대안) 확인 보안

0핵심 개념 — "컨테이너 안에서 설정하지 않는다"

봇 신원은 호스트에서 한 번 준비하면 컨테이너가 뜰 때 entrypoint가 자동으로 적용합니다. 컨테이너 안에서 손으로 git config를 칠 필요가 없습니다(마운트가 읽기전용이라 쓰면 실패하고, --rm이면 사라집니다).

신원은 두 부분입니다:

  • 커밋 작성자 — user.name / user.email
  • 푸시/풀 인증 — GitHub Personal Access Token(PAT) 또는 SSH deploy key

1준비 — 봇 계정 + PAT 발급

  1. 봇용 GitHub 계정(또는 최소 권한 계정) 준비

    개인 계정을 그대로 써도 되지만, 감사·권한 분리를 위해 전용 봇 계정 권장.

  2. PAT 발급 (권한 최소화)

    GitHub → Settings → Developer settings → Personal access tokens

    • Fine-grained(권장): 대상 저장소만 선택 → Repository permissions → Contents: Read and write (PR 만들려면 Pull requests: Read and write 추가)
    • Classic: repo 스코프

    토큰 문자열(ghp_... 또는 github_pat_...)을 복사해 둡니다. 커밋만 하고 푸시는 안 할 거면 토큰 없이 진행해도 됩니다(커밋 신원만 설정).

토큰은 비밀입니다. 저장소·이미지에 넣지 마세요. 아래 방식들은 모두 토큰을 저장소 밖(~/.agent)이나 런타임 env로만 다룹니다.

2호스트에 등록 (공유) 가장 추천

봇 신원을 호스트의 ~/.agent에 한 번 저장해두면, 이후 뜨는 모든 컨테이너가 자동으로 공유합니다(로그인·크레덴셜과 동일 원리). 한 번만 하면 되고, 컨테이너 안에서 아무것도 안 해도 됩니다. 아래 둘 중 편한 방식 — 결과(호스트에 gitconfig/git-credentials 생성)는 동일합니다.

2-A. agentpod CLI가 있으면 — 한 줄

# 호스트에서 1회
$ agentpod git-setup \
    --name "agent-bot" \
    --email "agent-bot@users.noreply.github.com" \
    --token <PAT>          # 생략 시 커밋 신원만 설정

# 이후 그냥 실행 — 신원이 자동으로 들어가 있음
$ cd 프로젝트 && agentpod run

2-B. Docker만 쓸 때 — 파일 직접 생성

# 호스트에서 1회 (2-A가 만드는 것과 같은 파일)
$ mkdir -p ~/.agent
$ cat > ~/.agent/gitconfig <<'EOF'
[user]
	name = agent-bot
	email = agent-bot@users.noreply.github.com
[safe]
	directory = *
[credential]
	helper = store
EOF
$ echo "https://x-access-token:<PAT>@github.com" > ~/.agent/git-credentials
$ chmod 600 ~/.agent/git-credentials

# 실행 시 읽기전용으로 마운트
$ docker run --rm -it -v "$PWD":/project -w /project \
    -v ~/.agent/claude:/home/agent/.claude \
    -v ~/.agent/gitconfig:/home/agent/.gitconfig:ro \
    -v ~/.agent/git-credentials:/home/agent/.git-credentials:ro \
    agentpod:latest claude --dangerously-skip-permissions

저장 위치(모든 컨테이너가 읽기전용으로 마운트):

파일내용
~/.agent/gitconfiguser.name/email + safe.directory (+ credential.helper)
~/.agent/git-credentials토큰 (chmod 600, 토큰 줬을 때만)
왜 이게 제일 추천인가: ①한 번만 — 프로젝트/실행마다 다시 안 넣어도 됨. ②모든 컨테이너 공유 — 로그인처럼 ~/.agent에 영속. ③토큰이 실행 명령·프로세스 목록에 노출되지 않음(env 방식은 명령에 토큰이 남음). ④git config에도 정상 표시.

3환경변수로 그때만 (대안·비영속) Docker만

호스트에 파일을 안 만들고 그 실행에만 신원을 넣고 싶을 때. 호스트에서 값을 env로 넘기면 entrypoint가 적용합니다.

$ docker run --rm -it -v "$PWD":/project -w /project \
    -v ~/.agent/claude:/home/agent/.claude \
    -e GIT_BOT_NAME="agent-bot" \
    -e GIT_BOT_EMAIL="agent-bot@users.noreply.github.com" \
    -e GITHUB_TOKEN="<PAT>" \
    agentpod:latest claude --dangerously-skip-permissions

.env 파일에 같은 키를 넣고 --env-file .env로 줘도 동일합니다.

한계: 그 실행에만 적용(공유 아님) · 커밋 신원을 git 환경변수로 넣어 git config user.name엔 안 보임(커밋엔 반영) · 토큰이 실행 명령/셸 히스토리에 남을 수 있음. → 지속·공유가 필요하면 §2 호스트 등록을 쓰세요.

5확인

# 컨테이너 안에서
agent@…$ git config --global user.name       # 호스트 등록 → agent-bot
agent@…$ echo "$GIT_AUTHOR_NAME"                # 환경변수 방식 → agent-bot

# 실제 커밋/푸시로 최종 확인
agent@…$ git commit -m "test" && git log -1 --format='%an <%ae>'
agent@…$ git push                            # 토큰이 있으면 봇으로 push
방식등록 위치공유git config에 표시
호스트 등록 · git-setup (권장)호스트 1회✅ 전 컨테이너✅
호스트 등록 · 파일 (권장, Docker만)호스트 1회✅ 마운트한 것✅
환경변수 (대안)docker run마다❌ 그 실행만❌ (커밋엔 반영)

6보안 체크리스트

  • 봇 계정/토큰 권한은 최소로(필요한 저장소만, Contents write 정도).
  • 토큰은 주기적으로 회전. 유출 시 즉시 폐기.
  • 토큰을 저장소·이미지·compose에 하드코딩하지 않기. .env는 gitignore, ~/.agent/git-credentials는 저장소 밖 + chmod 600.
  • 사람 개인키·호스트 gitconfig/ssh를 컨테이너에 넣지 않기(전용 봇 신원으로 대체).
