AgentPod

스킬(플러그인) 추가 가이드

프로젝트 루트에 매니페스트 파일을 두면, 컨테이너가 뜰 때 선언한 Claude Code 스킬(플러그인)을 자동 설치합니다.

skills.toml / agent.toml 프로젝트 루트 부팅 시 자동 설치 agentpod export로 공유
개요 ① 어디에 ② 스키마 ③ 이름만 (카탈로그) ④ 커스텀 스킬 ⑤ 적용·확인 ⑥ 공유하기 문제 해결

0개요

Claude Code 스킬은 플러그인으로 배포되고, 플러그인은 마켓플레이스(= git 저장소)에서 받습니다. AgentPod는 컨테이너 부팅 시 프로젝트 루트의 매니페스트를 읽어 claude plugin marketplace add + claude plugin install을 자동 실행합니다(멱등·best-effort — 실패해도 부팅은 계속).

1어디에 적나

프로젝트 루트(= 컨테이너 작업 디렉토리)에 아래 둘 중 하나(또는 둘 다 — 병합됨):

파일용도
skills.toml스킬 목록만 담을 때
agent.toml다른 설정과 함께 담을 때 ([[skills]] 섹션)
내-프로젝트/
├── skills.toml        # 또는 agent.toml
├── CLAUDE.md
└── ...

agents/verify·agents/verify/spec-inspect 같은 폴더도 각각 프로젝트 루트이므로, 거기에 두면 그 에이전트에만 적용됩니다.

2스키마

# skills.toml (또는 agent.toml)
[[skills]]
name = "superpowers"            # 필수: 플러그인 이름
source = "github:org/repo"       # 선택: 마켓플레이스 저장소 (카탈로그에 있으면 생략 가능)
marketplace_name = "..."        # 선택: install 시 @이름 (add 출력에서 자동 감지 시도)
ref = "main"                    # 선택: 브랜치/태그
enabled = true                    # 선택: 기본 true (false면 건너뜀)
필드필수설명
name✅플러그인 이름
source△마켓플레이스 저장소(github:org/repo 또는 org/repo). 카탈로그에 없는 스킬은 필수
marketplace_name—install 대상 name@marketplace의 마켓플레이스 이름. 생략 시 add 출력에서 자동 감지
enabled—false면 설치 안 함

3이름만으로 추가 (내장 카탈로그)

자주 쓰는 스킬은 이름만 적으면 됩니다. AgentPod 내장 카탈로그가 출처를 대신 채웁니다.

[[skills]]
name = "superpowers"

현재 카탈로그: superpowers → anthropics/claude-plugins-official (마켓플레이스 claude-plugins-official, 공식 마켓플레이스). 카탈로그에 없는 스킬은 §4처럼 source를 지정하세요.

4커스텀 스킬 추가

  1. 마켓플레이스 저장소 확인

    스킬 제공자의 GitHub 저장소(마켓플레이스)를 확인합니다. 대개 저장소에 .claude-plugin/marketplace.json이 있고, 거기에 마켓플레이스 이름과 포함 플러그인이 선언돼 있습니다.

  2. 매니페스트에 선언

    [[skills]]
name = "my-skill"
source = "github:org/my-skill-repo"
# marketplace_name = "my-market"   # marketplace.json의 name과 다르면 지정

  3. 이름 확인이 어렵다면

    marketplace_name을 비워두면 AgentPod가 marketplace add 출력에서 이름을 자동 감지해 name@감지된이름으로 설치를 시도합니다. 그래도 실패하면 로그에 skipped로 남습니다.

수동 확인 팁: 컨테이너 셸(agentpod shell)에서 claude plugin marketplace add org/repo 후 claude plugin marketplace list로 등록된 이름을, claude plugin install <tab>/문서로 플러그인 이름을 확인할 수 있습니다.

5적용 · 확인

# 매니페스트를 둔 프로젝트에서 실행하면 부팅 시 자동 설치
$ agentpod run          # (또는 docker run — 프로젝트가 마운트되면 동일)

# 컨테이너 안에서 설치 확인
agent@…$ claude plugin list
영속: 설치는 ~/.claude(= 호스트의 프로젝트별 격리 디렉토리에 바인드 마운트)에 남아 재실행에도 유지됩니다. 기본은 프로젝트마다 분리되어 있어 다른 프로젝트 컨테이너에는 자동으로 안 보입니다 — 다른 사람과 공유하려면 §7 공유하기를 참고하세요.

6공유하기 — agentpod export

세션 중에 설치한 플러그인/스킬은 프로젝트별로 분리된 상태에만 남아 git clone해도 자동으로 안 따라옵니다. agentpod export로 지금 상태를 agent.toml에 스냅샷해서 커밋하면 팀원과 공유됩니다.

자세한 사용법·시크릿 처리·문제 해결: 공유 가이드 참고.

7문제 해결

증상원인 / 해결
설치가 안 됨(skipped)마켓플레이스 이름이 안 맞음. marketplace_name을 명시하거나, agentpod shell에서 claude plugin marketplace add 출력의 이름 확인.
이름만 적었는데 안 됨카탈로그에 없는 스킬. source(마켓플레이스 저장소)를 추가하세요.
네트워크 오류마켓플레이스는 git clone으로 받습니다. 컨테이너 아웃바운드 네트워크 필요.
매니페스트가 안 읽힘파일이 프로젝트 루트(컨테이너 작업 디렉토리)에 있는지 확인. agent.toml/skills.toml 철자·TOML 문법 확인.
비활성화하고 싶음해당 항목에 enabled = false.
agentpod export 관련 문제공유 가이드 §6 참고.
